# Venturi v1 Implementation Plan

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build a usable Venturi v1 Linux audio mixer with PipeWire virtual devices, channel-based routing, GTK4/libadwaita UI, persistent categorization, soundboard, and global hotkeys.

**Architecture:** Implement in vertical slices on top of a small Rust core. First establish foundation (CLI/config/state/event bus), then bring up PipeWire graph and routing, then wire UI widgets to core events, then add mic/soundboard/hotkeys and package metadata. Keep PipeWire proxies fully isolated on a dedicated thread and communicate by typed command/event channels.

**Tech Stack:** Rust, pipewire-rs, gtk4-rs, libadwaita-rs, serde/toml, crossbeam-channel, symphonia, tracing, clap, ashpd, global-hotkey.

---

## File Structure Map (target)

- `Cargo.toml` — dependencies, features, package metadata
- `src/main.rs` — entrypoint, CLI args, logger, process mode (`--daemon`)
- `src/app.rs` — shared app state and command/event wiring
- `src/core/*` — PipeWire lifecycle, routing, volume, meters, hotkeys integration boundary
- `src/config/*` — config/state schema + persistence (XDG config/state split)
- `src/categorizer/*` — static map + metadata heuristics + persistent overrides
- `src/gui/*` — window shell and widgets (mixer, app chips, soundboard, settings)
- `src/audio/*` — noise gate + soundboard decode/mix logic
- `src/tray.rs` — tray icon integration
- `tests/*` — unit/integration tests for config/categorizer/volume mapping
- `data/*` — desktop metadata
- `flatpak/org.venturi.Venturi.json` — Flatpak packaging manifest

## Risks / Watchpoints

- PipeWire session-manager interactions (WirePlumber) may override routing if metadata path is incomplete.
- GTK DnD payload contracts can break silently if not strongly typed and tested.
- Flatpak permissions are necessary for direct PipeWire core registry operations.
- Keep `pipewire-rs` objects strictly on PW thread; never leak proxies across channel boundaries.
